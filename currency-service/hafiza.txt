## 1. SERVİSİN AMACI
Döviz kuru bilgilerini sağlayan servistir. İki farklı kullanım senaryosu
vardır: (1) Vitrin Modu — Redis cache'ten hızlı kur bilgisi sunma
(Frontend için), (2) Canlı Bypass Modu — Cache'i atlayarak anlık kur
çekme (havale/transfer işlemlerinde doğru çevrim için).

Veritabanı yoktur. Tek bağımlılık Redis'tir (cache).

## 2. ENDPOINTS (CONTROLLER UÇ NOKTALARI)

### CurrencyController (/api/v1/currencies)
| HTTP   | Endpoint  | Girdi                           | Çıktı               | Açıklama                         |
|--------|-----------|---------------------------------|----------------------|----------------------------------|
| GET    | /rates    | base (Query, default="TRY")     | ExchangeRateResponse | Kur bilgisi getir (Cache'li)     |
| GET    | /convert  | from, to, amount (Query Params) | Double (200 OK)      | Döviz çevirimi (Cache bypass)    |

## 3. GİRDİ VE ÇIKTILAR (DTO İÇERİKLERİ)

### Response DTO'ları
ExchangeRateResponse:
  - base_code : String (Taban para birimi, Örn: "TRY")
  - rates     : Map<String, Double> (Örn: {"USD": 0.029, "EUR": 0.027, "TRY": 1.0})

## 4. KUR KAYNAĞI VE CACHE MEKANİZMASI

### Hardcoded Kurlar (CurrencyServiceImpl)
Kaynak: Servis içinde sabit Map olarak tanımlı (Gerçek API entegrasyonu yok)
```
TRY -> {TRY: 1.0, USD: 0.029, EUR: 0.027}
USD -> {USD: 1.0, TRY: 34.5, EUR: 0.92}
EUR -> {EUR: 1.0, TRY: 37.0, USD: 1.09}
```

### Redis Cache (İki Yüzlü Strateji — "Two-Faced Cache")

#### 🟢 Vitrin Modu (/rates — @Cacheable)
- Cache Key Pattern: "exchangeRates::{baseCurrency}"
- TTL: RedisCacheConfig'te tanımlı (varsayılan serileştirici ile)
- İlk istekte hesaplanır, sonraki isteklerde Redis'ten döner
- Amaç: Frontend kur gösterimlerini hızlı servis etmek

#### 🔴 Bypass Modu (/convert — Cache YOK)
- @Cacheable YOKTUR, her çağrıda hesaplama yapılır
- Amaç: Havale/Transfer sırasında anlık doğru kur kullanmak
- Arka plandaki servisler (Backend) bu endpoint'i Feign ile çağırır

### RedisCacheConfig
- @EnableCaching aktif
- RedisCacheManager oluşturulur
- Jackson2JsonRedisSerializer ile JSON serileştirme
- TTL süresi framework tarafından yönetilir

## 5. İŞ KURALLARI (BUSINESS LOGIC)

### Kur Sorgulama (/rates)
1. base parametresi alınır (varsayılan: TRY)
2. rates Map'inden ilgili para birimi kurları çekilir
3. Bulunamazsa: BankOperationException("Desteklenmeyen para birimi: ...")
4. ExchangeRateResponse döner

### Döviz Çevirimi (/convert)
1. from parametresi ile rates Map'inden kur alınır
2. Bulunamazsa: BankOperationException("Desteklenmeyen para birimi: ...")
3. rates.get(toCurrency) ile hedef para birimi kuru alınır
4. Kur bulunamazsa: BankOperationException ("Desteklenmeyen para birimi: ...")
5. result = amount * rate hesaplanır ve döner
