## 1. SERVİSİN AMACI
Sistemin döviz kur bilgisi ihtiyacını karşılayan bağımsız mikroservistir.
"İki Yüzlü Redis Önbelleği" (Two-Faced Redis Cache) mimarisiyle çalışır:
- Frontend/Vitrin için: 10 dakika Redis önbellekli kurlar (performans)
- Para transferi için: Her seferinde dış API'ye giden canlı kurlar (doğruluk)

Veritabanı yoktur. Herhangi bir tabloya sahip değildir.
Dış sağlayıcı: https://open.er-api.com/v6/latest/{base}

## 2. VERİTABANI
YOK — Bu servisin veritabanı yoktur.
Tüm veriler anlık olarak dış API'den çekilir.
Redis sadece önbellek (cache) amacıyla kullanılır, kalıcı veri deposu değildir.

## 3. ENDPOINTS (CONTROLLER UÇ NOKTALARI)

### CurrencyController (/api/v1/currencies)
| HTTP | Endpoint | Girdi                    | Çıktı                | Açıklama                          |
|------|----------|--------------------------|----------------------|-----------------------------------|
| GET  | /rates   | base (Query Param: String)| ExchangeRateResponse | Döviz kurlarını getir (önbellekli)|

Örnek istek: GET /api/v1/currencies/rates?base=TRY
Tüketici   : Frontend (Next.js vitrin ekranı) + Bank-Demo-Backend (Feign üzerinden)

## 4. GİRDİ VE ÇIKTILAR (DTO İÇERİKLERİ)

### Response DTO'ları
ExchangeRateResponse (implements Serializable - Redis için zorunlu):
  - base_code : String (Örn: "TRY")
  - rates     : Map<String, Double> (Örn: {"USD": 0.03, "EUR": 0.028, "TRY": 1.0})

serialVersionUID = 1L (Redis byte dönüşümü için)

## 6. REDIS ÖNBELLEK KONFİGÜRASYONU (RedisCacheConfig)

```
Cache Adı  : "rates"
Cache Key  : baseCurrency (Örn: "TRY", "USD", "EUR")
TTL (Ömür) : 10 dakika (entryTtl = Duration.ofMinutes(10))
Key Serializer   : StringRedisSerializer (okunabilir string anahtarlar)
Value Serializer : GenericJackson2JsonRedisSerializer (JSON formatında değerler)
```

## 7. İŞ KURALLARI (BUSINESS LOGIC) — İKİ YÜZLÜ ÖNBELLEK MİMARİSİ

### Yüz 1 — VİTRİN OPERASYONU (getLiveRates — ÖNBELLEKLİ)
Tetikleyen: GET /api/v1/currencies/rates endpoint'i (Frontend ekranı)
- @Cacheable(value="rates", key="#baseCurrency") annotasyonu aktiftir
- İlk istekte dış API çağrılır, yanıt Redis'e 10 dk için yazılır
- Sonraki isteklerde Redis'ten okur, dış API çağrılmaz
- 10 dakika sonra TTL dolunca cache otomatik temizlenir, sonraki istekte API tekrar çağrılır

### Yüz 2 — KARARGAH OPERASYONU (convertAmount — ÖNBELLEKSİZ CANLI)
Tetikleyen: Bank-Demo-Backend → CurrencyServiceClient → GET /rates?base=..
- @Cacheable annotasyonu YOKTUR
- Doğrudan fetchRatesFromExternalApi() private metodunu çağırır
- getLiveRates() metodu ÇAĞRILMAZ (cache bypass)
- Her transfer işleminde dış API'ye gider ve gerçek anlık kur kullanılır
- Aynı para birimi kontrolü: fromCurrency == toCurrency ise dış API'ye gidilmez

### Neden İki Yüz?
- Vitrin (Frontend): Kullanıcı kur tablosunu her saniye yenilemez, 10 dk önbellek
  performansı artırır ve dış API'ye gereksiz istek atılmaz
- Transfer: Para transferinde 10 dk önceki kur kullanılırsa müşteri zarar görebilir.
  Bu nedenle transferde her zaman anlık kur zorunludur
- Bu desen "Two-Faced Redis Cache Bypass" olarak sistemin mimarisinde tanımlanmıştır

### Döviz Çevirme Akışı (convertAmount)
1. fromCurrency == toCurrency → Direkt amount döner (API çağrısı YOK)
2. fetchRatesFromExternalApi(fromCurrency) ile dış API'den kurlar çekilir
3. rates.get(toCurrency) ile hedef para birimi kuru alınır
4. Kur bulunamazsa: BankOperationException ("Desteklenmeyen para birimi: ...")
5. result = amount * rate hesaplanır ve döner
