# 🚪 API GATEWAY (PORT 8081) HAFIZA KARTI

Tüm sistemin dış dünyaya açılan uç beygiri, "Zero Trust" prensibinin mutasyon operatörü ve tek giriş merkezidir. Veritabanı yoktur, konfigürasyonu en kritik olan katmandır.

## 🏗 1. KİMLİK GÜMRÜĞÜ (KEYCLOAK OAUTH2 RESOURCE SERVER)

Projede lokal bir JWT parser (`JwtUtil`) veya custom authentication filter **YOKTUR**. Filtre altyapısını bütünüyle Spring Security Oauth2 Resource Server ve `Keycloak` yönetir.

### 1.1. `SecurityConfig.java` Rolü
*   **Public (Açık) Kapılar:** `/api/v1/auth/register` (kayıt), `/api/v1/currencies/rates` (ziyaretçi kur tarama) ve `/eureka/**` dışında gelen akımsız tüm dış trafik reddedilir.
*   **Tercüman Devrede (`grantedAuthoritiesExtractor`):** Keycloak sisteminden Bearer token ulaştığında, Gateway doğrudan bu JWT biletindeki şifreyi kırar. İçerisindeki kalabalık rollerden `realm_access.roles` objesine ulaşıp `["ADMIN", "RETAIL_CUSTOMER"]` gibi yetkilerin önüne **`ROLE_`** basarak Spring'in okuyabileceği `SimpleGrantedAuthority` nesnesine kalibre eder.

### 1.2. Mimari Mutasyon: `KeycloakHeaderFilter.java`
Bu Global Süzgeç Sınıfı projenin Zero Trust operasyonunun kalbidir. Sistemin çalışması şu şekildedir:
1.  SecurityContext'ten doğrulanmış Token (`Jwt`) çekilir.
2.  `preferred_username` iddialarından (class) kişinin net kimlik numarası (TC) olan `identityNumber` parse edilir.
3.  Kişinin bütün yetki statüleri virgüllü stringe dönüştürülür.
4.  Gateway **Trafiği Mutasyona Sokar:** İsteğin `Headers` kısmının içine kilit vurarak `X-Identity-Number` ve `X-User-Role` değişkenlerini gömer.
5.  **💣 Yıkıcı Emrah Hamlesi:** Gateway arka tarafa asla güvenilmeyen sızıntı vermemek için gelen istekteki koskoca `Authorization: Bearer <TOKEN>` başlığını tamamen siler! Arka mikroservislere sadece ve sadece Gateway'in tasdikleyip mühür vurduğu "X-Header" bilgileri iletilir. Bu sayede hiçbir alt servis token çözmekle zerre kadar uğraşmaz.

## 🧭 2. YÖNLENDİRME TRAFİĞİ (`application.yml` veya RoutingConfig)

Filtreden geçen veya public olan akış, hedef servisin ismine (`LoadBalancer`) vuracak biçimde tünellenir.
1.  **DÖVİZ:** `Path=/api/v1/currencies/**` $\rightarrow$ `lb://currency-service`
2.  **FATURA:** `Path=/api/v1/bills/**` $\rightarrow$ `lb://bank-bill-micro`
3.  **BİREYSEL/ADMIN/KİMLİK PROFİLLERİ:** `/api/v1/auth/**`, `/api/v1/customers/**`, `/api/v1/admin/customers/**` $\rightarrow$ `lb://auth-service`
4.  **MEMUR VE ŞİRKETLER:** `/api/v1/companies/**` $\rightarrow$ `lb://bank-corporate-micro`
5.  **KARARGAH (HESAP/İŞLEM):** `/api/v1/**` (Son tünel, Regex Fallback) kapsamına giren tüm Account (`/accounts`) ve Transaction (`/transactions`) işleri $\rightarrow$ `lb://bank-demo-backend`

Bu haritalandırmaları Eureka DNS listesine sorarak çözer ve IP kullanmadan (e.g., `http://discovery-server:8761/eureka/` adresinden bakarak) pürüzsüz iletir.
