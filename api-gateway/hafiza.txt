# 🚪 SEC-DEMO API GATEWAY HAFIZA KARTI (V4.0 - EXTREME DETAYLI)

Bütün trafik yönetiminin, güvenlik duvarı (`Authorization` filter) kontrollerinin yapıldığı giriş noktasıdır. Veri tabanı barındırmaz.

---

## 🏗️ 1. MİMARİ VE EUREKA
*   **Çekirdek:** Spring Cloud Gateway (Port 8081).
*   **Eureka Client:** `http://localhost:8761/eureka/` (veya `$EUREKA_URL`) adresine kendini kaydeder.

## 🛡️ 2. FİLTRELER VE CORS (GÜVENLİK AYARLARI KODU)

### 2.1. CORS Sınırı (`CorsConfig.java`)
Browser tabanlı Pre-Flight (`OPTIONS`) isteklerinin havada yakalanıp geçirilmesini sağlar. Ayarlar `CorsWebFilter` `registerCorsConfiguration("/**", corsConfig)` üzerinden tüm projeye vurur:
*   `allowedOrigins`: Hata vermemesi için `allowedOriginPatterns: "*"` (Herkese açık yapılmıştır veya array olarak geçilmiştir, kodda `localhost:3000` test izni de mevcuttur).
*   `allowedMethods`: `"GET", "POST", "PUT", "DELETE", "OPTIONS"`.
*   `allowedHeaders`: `*`.
*   `allowCredentials`: `true` (Bearer Token taşımak için zorunludur).

### 1.1. Ağ Ayarları ve Bağımlılıklar
*   Uygulama **8081** portunda ayağa kalkar. Dış dünyadan gelen ve Frontend'in gönderdiği tüm "fetch" veya "axios" istekleri direkt buraya çarpar.
*   **Eureka Client (`discovery-server:8761`)**: Tüm mikroservislerin ismini IP'ye çevirmek üzere `discovery-server` DNS havuzuna bağlanır. `application.yml` dosyasında Eureka url'i explicitly "http://discovery-server:8761/eureka/" olarak güncellenmiştir (Docker tam uyumluluğu için).
*   Ortak 512-bit gizli `jwt.secret` anahtarını application config içerisinde barındırır. Auth servisle aynı anahtara sahip olduğu için şifreyi çözebilir.

### 2.2. Oauth2 ve Keycloak Resource Server (`SecurityConfig.java`)
Eksiden kullanılan lokal JWT doğrulama işi çöpe atıldı. Gateway artık Keycloak entegreli bir `oauth2ResourceServer` olarak çalışır:
*   Açık rotalar haricindeki HER istek için `authenticated()` (Token ile korunmuş) kuralı vardır.
*   **Keycloak Tercüman Motoru (`grantedAuthoritiesExtractor`):** Gateway kendisine Bearer olarak sunulan Keycloak JWT Biletinin (`realm_access.roles`) kasasını bizzat kırar ve içindeki rolleri okuyarak önüne "ROLE_" takısı koyar ve Spring'in anlayacağı yetki paketine döndürür.

### 2.3. Mimarinin Kalbi: Keycloak Başlık Filtresi (`KeycloakHeaderFilter.java`)
Bütün mimarinin kaderini değiştiren Global Süzgeç!
1. Gateway dış dünyadan gelen Keycloak biletini doğrular, içinden `preferred_username` claim'ini yani **TC Kimlik Numarasını** (`identityNumber`) çıkarır.
2. Rolleri bir string halinde virgülle birleştirir.
3. **KRİTİK HAMLE (MUTASYON):** Gelen isteği mutasyona uğratarak içine `X-Identity-Number` ve `X-User-Role` HTTP Header'larını çakar!
4. **Zero Trust Bypass:** İsteğin üzerindeki orjinal Token'ı (`Authorization: Bearer...`) **SİLER**. Arkadaki Karargaha veya corporate servislere Token gitmez. O servisler jwt çözmekle uğraşmaz, Gateway'in sözüne (Header) güvenerek lokal filtreleriyle (`InternalHeaderFilter`) hayatına devam eder.

## 🔀 3. ROUTING REHBERİ (`GatewayRoutingConfig.java` V5.2 STANDARDI)
Prensip: Yukarıdaki kurallar eşleşmezse en alttaki Monolith Karargaha düşer.

1.  **Döviz (`currency-service-route`):** `/api/v1/currencies/**` hedefini yakalar, `http://currency-service:8083` servis adresine zıplatır. (Polis filtresi yok, PermitAll).
2.  **Fatura (`bill-service-route`):** `/api/v1/bills/**` hedefini yakalar, `http://bill-service:8084` sunucusuna zıplatır. (Auth Filter Aktif).
3.  **Kurumsal Yönetim (`corporate-service-route`):** `/api/v1/companies/employees/**` rotasını yakalar, `http://corporate-service:8085` sunucusuna zıplatır. (Auth Filter Aktif).
4.  **Admin Müşteri Hesapları (`backend-admin-customer-accounts-route`):** `/api/v1/admin/customers/*/accounts` hedefine gidiyorsa, hesap işlemlerini Monolith yapacağı için istisnai olarak `http://backend:8080`'a zıplatılır.
5.  **Admin Kullanıcı İşlemleri (`auth-admin-route`):** `/api/v1/admin/customers/**` (Kullanıcı listeleme, onaylama) işlemleri `http://auth-service:8086` hedefine zıplatılır.
6.  **Kimlik Üssü (Auth & Profile):** `/api/v1/auth/**` ve `/api/v1/customers/**` çağrıları login ve profil işlemleri için `http://auth-service:8086` sunucusuna aktarılır.
7.  **Karargah (Monolith):** `/api/v1/**` (Tüm Accounts, Transactions ve Internal mantıklar) `http://backend:8080` adresine vurur. (Auth Filter var).
