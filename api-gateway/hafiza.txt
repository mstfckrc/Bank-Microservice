# 🚪 ABSOLUTE BLUEPRINT: api-gateway (PORT 8081)

Bu belge, API Gateway (Trafik Yönlendirici) servisinin src/main/java/com/mustafa altındaki TÜM klasörlerini ve sınıflarını listeler. Bu mikroservis MVC (Veritabanı, Entity, Service) taşımaz, sadece konfigürasyon ve filtrelerden oluşur.

---

## 1. PAKET: `starter`
*   **`ApiGatewayApplication.java`**: `@SpringBootApplication`, `@EnableDiscoveryClient`. Sistemin ana giriş kapısını ayağa kaldırır.

---

## 2. PAKET: `config` (Altyapı Ayarları)
*   **`GatewayRoutingConfig.java`**: Spring Cloud Gateway'ın `RouteLocator` bean'ini tanımlar. Hangi yolun hangi servise düşeceğini kod üzerinden (`.route(p -> p.path("/api/v1/auth/**").uri("lb://auth-service"))`) hardcoded veya `.yml` olarak bağlar.
*   **`CorsConfig.java`**: Front-end'den (React/Next.js) gelen tarayıcı erişim engellerini kaldırmak için `CorsWebFilter` tanımlar (Tüm Originlere, Headerlara, Metotlara izin verilir `*`).
*   **`SecurityConfig.java`**: Spring Security WebFlux konfigürasyonudur (`ServerHttpSecurity`).
    *   OAuth2 Resource Server modunda çalışır (`jwt()`).
    *   `/api/v1/auth/**` dışa açılır (`permitAll()`).
    *   Geri kalan her şey zorunlu kimlik doğrulamasına tabidir (`authenticated()`).

---

## 3. PAKET: `filter` (Trafik Kalkanları)
*   **`KeycloakHeaderFilter.java`**: (`implements GlobalFilter, Ordered`) **EN KRİTİK SINIF**.
    *   Gateway'den arka servislere geçen her HTTP isteğini havada yakalar.
    *   İsteğin içindeki OAuth2 / JWT kılıfından Keycloak bilgilerini parçalar.
    *   Token içindeki `preferred_username` id'sini yakalar ve arkaya giden yeni isteğin `X-Identity-Number` kutusuna çakar!
    *   Aynı şekilde Token'da dönen rolleri `X-User-Role` string'i olarak yapıştırır.
    *   Dışarıdan gelen ana `Authorization` token bilgisini arkaya göndermemek için **SİLER/TEMİZLER (Zero-Trust Bypass).** Çünkü JWT çözümleme yükü sadece Gateway'e aittir, arka mikroservisler güvende olacak.
