## 1. SERVİSİN AMACI
Tüm dış trafiğin tek geçiş noktasıdır. Spring Cloud Gateway (reactive /
WebFlux tabanlı) üzerinde çalışır. Üç temel görevi vardır:
1. JWT Token doğrulaması (Keycloak ile OAuth2 Resource Server)
2. İstekleri arkadaki mikro servislere yönlendirme (Routing)
3. JWT claim'lerini mikroservislerin anlayacağı header'lara dönüştürme
   (X-Identity-Number, X-User-Role) ve Authorization header'ını silme

Veritabanı yoktur. Redis sadece Rate Limiter için kullanılır.

## 2. GÜVENLİK MİMARİSİ — İKİ KAPILI SİSTEM (SecurityConfig)

### 🟢 Kapı 1: SİVİL BÖLGE (@Order(1) — Öncelik 1)
Token doğrulaması YAPILMAZ. Aşağıdaki pattern'lar bu kapıdan geçer:

| Pattern                     | Yöntem    | Açıklama                    |
|-----------------------------|-----------|---------------------------  |
| /**                         | OPTIONS   | CORS Preflight serbest geçiş|
| /api/v1/auth/**             | Tümü      | Kayıt, giriş, captcha       |
| /api/v1/currencies/**       | Tümü      | Kur vitrin endpoint'i       |
| /eureka/**                  | Tümü      | Servis keşif UI'ı           |

Kritik NOT: Bu kapıda `.oauth2ResourceServer()` YOKTUR.
Böylece süresi dolmuş token ile gelen istek hata vermez —
sadece token doğrulanmaz.

### 🔴 Kapı 2: ASKERİ BÖLGE (@Order(2) — Varsayılan Kapı)
Sivil bölgeye giremeyen her istek buraya düşer. Token ZORUNLUDUR.
- `.authorizeExchange -> anyExchange().authenticated()`
- `.oauth2ResourceServer -> jwt -> grantedAuthoritiesExtractor()`

### 🕵️ Keycloak Tercümanı (grantedAuthoritiesExtractor)
JWT'nin `realm_access.roles` claim'ini Spring Security Authority formatına çevirir:
- "ADMIN"           → "ROLE_ADMIN"
- "RETAIL_CUSTOMER" → "ROLE_RETAIL_CUSTOMER"
- "CORPORATE_MANAGER" → "ROLE_CORPORATE_MANAGER"
Claim yoksa veya boşsa Collections.emptyList() döner (yetkisiz).

## 4. JWT → HEADER DÖNÜŞÜM FİLTRESİ (KeycloakHeaderFilter)
implements GlobalFilter, Ordered (order = -1, yetki kontrolünden hemen sonra)

Akış:
1. ReactiveSecurityContextHolder'dan JWT alınır
2. jwt.getClaimAsString("preferred_username") → identityNumber (TC/Vergi No)
3. authentication.getAuthorities() → roller "," ile birleştirilir
4. İstek mutate edilir:
   - headers.set("X-Identity-Number", identityNumber)  ← EKLENİR
   - headers.set("X-User-Role", roles)                 ← EKLENİR
   - headers.remove("Authorization")                   ← SİLİNİR ‼️

KRİTİK: Authorization header'ı (JWT) arkadaki servise ASLA iletilmez.
Arkadaki servisler sadece sade header'ları (X-Identity-Number) görür.
Bu sayede mikroservisler Keycloak'a bağımlı olmaz.

## 5. HIZLANMA LİMİTİ (RateLimitConfig — Kayıt Bombalamasına Karşı Kalkan)

Sadece `/api/v1/auth/register` endpoint'ine uygulanır.

| Parametre       | Değer   | Açıklama                                       |
|-----------------|---------|------------------------------------------------|
| replenishRate   | 1       | Saniyede 1 token üretilir (sürekli akış hızı)  |
| burstCapacity   | 2       | Anlık en fazla 2 istek patlamasına izin verilir|
| KeyResolver     | IP Adresi| Her IP adresi ayrı sayaç tutar (Redis'te)      |

İmplementasyon: `RedisRateLimiter(1, 2)` — Redis tabanlı, dağıtık sayaç
KeyResolver: `exchange.getRequest().getRemoteAddress().getAddress().getHostAddress()`
IP alınamazsa "unknown" anahtarı kullanılır.

## 6. CORS KONFİGÜRASYONU (CorsConfig)
CorsWebFilter ile tüm /** path'lerine uygulanır.

| Ayar                  | Değer                              |
|-----------------------|------------------------------------|
| AllowedOriginPatterns | * (tüm origin'ler)                 |
| AllowedMethods        | GET, POST, PUT, DELETE, OPTIONS    |
| AllowedHeaders        | * (tüm header'lar)                 |
| AllowCredentials      | true (Token ve Cookie taşınabilir) |

## 7. MİMARİ BÜTÜNLÜK NOTU
Gateway, Nginx'in arkasındadır. Trafik akışı:
  İnternet → Nginx (HTTPS/SSL, 443) → api-gateway (8082) → Mikroservisler

Gateway'in Eureka'ya kaydı vardır, ancak servisler Gateway adını
Eureka üzerinden bulmak yerine Docker servis adlarıyla (hostname) haberleşir.
Routing'de `lb://` (Load Balancer) yerine doğrudan `http://servis-adı:port`
kullanılır — bu Docker ağı içindeki DNS çözümlemesini sağlar.
