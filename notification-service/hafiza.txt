## 1. SERVİSİN AMACI
Sistemdeki tüm bildirimlerin tek tüketicisidir (Consumer). Diğer
servisler (Auth, Backend, Corporate, Bill) RabbitMQ'ya mesaj gönderir,
bu servis kuyruğu dinleyerek bildirimleri işler. Şu an e-posta gerçekten
gönderilmez — sadece konsola loglanır. Gerçek SMTP entegrasyonu için
JavaMailSender veya harici API (SendGrid, Mailgun) eklenebilir.

Veritabanı YOKTUR. Herhangi bir Feign Client bağlantısı YOKTUR.

## 2. MESAJ TÜKETİM MEKANİZMASI

### NotificationListener
@RabbitListener(queues = "notification_queue")
receiveNotification(NotificationMessage message) metodu:

Dinlediği mesaj yapısı:
```
NotificationMessage:
  - destination       : String (e-posta adresi, telefon no vb.)
  - subject           : String (bildirim başlığı)
  - content           : String (bildirim içeriği)
  - identityNumber    : String (kime ait olduğu)
  - notificationType  : NotificationType enum
```

### NotificationType Enum
| Değer              | Açıklama                    | Örnek Tetikleyici                  |
|--------------------|-----------------------------|------------------------------------|
| EMAIL              | E-posta bildirimi           | Hesap açıldı, profil güncellendi   |
| SMS                | SMS bildirimi               | (Gelecek entegrasyon)              |
| PUSH_NOTIFICATION  | Anlık bildirim              | Para yatırıldı                     |
| SYSTEM_ALERT       | Sistem uyarısı (Admin'e)    | MASAK limiti aşıldı, itiraz geldi  |

### Mesaj İşleme Akışı
1. RabbitMQ kuyruğundan mesaj alınır (Jackson2JsonMessageConverter ile deserialize)
2. notificationType'a göre dallanma:
   - EMAIL: Konsola "[E-POSTA GÖNDERİLİYOR]" loglanır
   - SMS: Konsola "[SMS GÖNDERİLİYOR]" loglanır
   - PUSH_NOTIFICATION: Konsola "[PUSH BİLDİRİMİ]" loglanır
   - SYSTEM_ALERT: Konsola "[SİSTEM ALARMI]" loglanır
3. Tüm durumlarda mesaj detayları konsola yazdırılır (destination, subject, content)

## 3. KONFIGURASYON

### RabbitMQ Ayarları (RabbitMQConfig)
- Queue      : "notification_queue" (durable=true, eksik ise oluşturulur)
- Exchange   : "bank_exchange" (TopicExchange, durable=true)
- Routing Key: "notification_routing_key"
- Binding    : Queue <-> Exchange (routing key ile bağlanır)
- Converter  : Jackson2JsonMessageConverter (JSON otomatik parse)
Mesaj Format: JSON (Jackson2JsonMessageConverter)

### application.yml
```yaml
server:
  port: 8082
spring:
  application:
    name: notification-service
  rabbitmq:
    host: ${SPRING_RABBITMQ_HOST}
    port: ${SPRING_RABBITMQ_PORT}
    username: ${SPRING_RABBITMQ_USERNAME}
    password: ${SPRING_RABBITMQ_PASSWORD}
```
