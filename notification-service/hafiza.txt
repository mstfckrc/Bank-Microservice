# 📯 SEC-DEMO NOTIFICATION (BİLDİRİM) SERVİSİ HAFIZA KARTI (V4.0 - EXTREME DETAYLI)

Sistemin olay güdümlü asenkron bildiricisidir. Sadece RabbitMQ kuyruğundan mesaj çekip loglar ve simüle eder. REST API (Controller) ve DB barındırmaz.

---

## 🏗️ 1. MİMARİ
*   **Çekirdek:** Spring Boot. Port: 8082. `notification-service` adıyla Eureka'ya bağlı.
*   **MQ Bağlantısı:** `localhost:5672` (RabbitMQ).

## 📦 2. VERİ TRANSFER BİLEŞENLERİ (DTO MAPPINGS)
Gelen JSON mesajı `@RabbitListener` üzerinde `Jackson2JsonMessageConverter` sayesinde doğrudan Java objesine çevrilmektedir.

### 2.1. `NotificationMessage` (Gelen DTO)
Bu DTO, ana backend projesindeki yapıyla %100 birebir olmalıdır.
*   **Alanlar (Fields):**
    *   `String identityNumber` (Müşteri TC No)
    *   `String destination` (Email/Tel vs)
    *   `String subject` (Mail konusu vs)
    *   `String content` (Gönderilecek uzun yazı)
    *   `NotificationType notificationType` (Enum Tipi).

### 2.2. `NotificationType` (Enum)
*   **Değerler:** `EMAIL`, `SMS`, `PUSH_NOTIFICATION`, `SYSTEM_ALERT`.

## ⚙️ 3. DİNLEYİCİ MANTIĞI (LISTENER)
*   **Sınıf:** `NotificationListener`
*   **Kuyruk Nöbeti:** `@RabbitListener(queues = RabbitMQConfig.NOTIFICATION_QUEUE)` ("notification_queue") kuyruğunu tutar.
*   **Çalışma Ritmi:**
    1.  Mesaj gelir gelmez Terminale `[POSTANE NÖBETÇİSİ UYANDI]` logunu atar.
    2.  `switch (notification.getNotificationType())` ile Enum tipine göre farklı log bloklarına (`Email gönderiliyor...` vb) girer.
    3.  `Thread.sleep(2000)` ile işlemi 2 tam saniye bekleterek asenkron çalışmayı ağrılı gösterir (Test amaçlı yapay yük).
    4.  `[✅ GÖREV TAMAM: Bildirim iletildi.]` logunu üretir ve işlemi bitirir. Hata olursa `InterruptedException` yakalayıp `error` loguna basar.
    5.  Mesaj kuyruktan silinir.
