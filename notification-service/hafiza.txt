## 1. SERVİSİN AMACI
Sistemdeki tüm servislerin asenkron bildirim ihtiyacını karşılayan
merkezi postane servisidir. REST API endpoint'i yoktur. Sadece RabbitMQ
kuyruğunu dinler ve gelen mesajları tipine göre işler.

NotificationMessage göndererek bu servisi tetikler. Bu sayede bildirim
göndermek isteyen servis işlemini bitirmeden beklemek zorunda kalmaz —
asenkron, decoupled (bağımsız) mesajlaşma.

Veritabanı yoktur. HTTP endpoint'i yoktur.

## 5. GİRDİ VE ÇIKTILAR (MESAJ DTO)

### NotificationMessage (implements Serializable)
| Alan            | Tip              | Açıklama                                      |
|-----------------|------------------|-----------------------------------------------|
| destination     | String           | Kime gidecek (e-posta, telefon, kimlik no)    |
| subject         | String           | Mesaj başlığı / konusu                        |
| content         | String           | Asıl mesaj metni                              |
| identityNumber  | String           | Maskeli TC (log/trace için)                   |
| notificationType| NotificationType | EMAIL / SMS / PUSH_NOTIFICATION / SYSTEM_ALERT|

NotificationType Enum:
  - EMAIL             : E-posta bildirimi
  - SMS               : SMS bildirimi
  - PUSH_NOTIFICATION : Mobil uygulama push bildirimi
  - SYSTEM_ALERT      : Admin paneline veya loglara kritik uyarı

## 6. İŞ KURALLARI (BUSINESS LOGIC)

### Mesaj Alım Akışı (NotificationListener)
@RabbitListener(queues = "notification_queue")
1. RabbitMQ'dan JSON mesaj alınır
2. Jackson2JsonMessageConverter, JSON'ı NotificationMessage DTO'ya dönüştürür
3. notificationType değerine göre switch-case dağıtım yapılır:

| Tip                | Eylem                                             |
|--------------------|---------------------------------------------------|
| EMAIL              | E-posta gönderim simülasyonu (log atılır)         |
| SMS                | SMS gönderim simülasyonu (log atılır)             |
| PUSH_NOTIFICATION  | Mobil push bildirim simülasyonu (log atılır)      |
| SYSTEM_ALERT       | Kritik uyarı log simülasyonu (log atılır)         |

4. Thread.sleep(2000) ile gönderim süresi simüle edilir
5. "Bildirim iletildi. Zarf imha edildi!" logu atılır

NOT: Şu an tüm bildirim tipleri sadece LOG'a yazılır (simülasyon).
Gerçek sistemde buradaki switch-case içine JavaMailSender (email),
Twilio (SMS), Firebase (push), PagerDuty (alert) entegrasyonları eklenir.
