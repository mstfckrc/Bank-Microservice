## 1. SERVİSİN AMACI
Sistemin finansal operasyonlarının tamamını yürüten ana bankacılık
servisidir. Hesap açma/kapatma, para yatırma, transfer (EFT/Havale),
toplu maaş ödemesi ve fatura tahsilatı bu serviste işlenir. 50.000 TL
ve üzeri transferler MASAK uyum mekanizması ile PENDING_APPROVAL
statüsüne alınır ve Admin onayına gönderilir. Elasticsearch üzerinden
sistem log sorgulama ve cAdvisor/Prometheus ile entegre monitoring
desteği mevcuttur.

## 2. VERİTABANI VE ENTITY MODELLERİ
Veritabanı: bank_core_db (PostgreSQL 15, Port: 5432)

### Tablo: accounts
| Kolon               | Tip             | Kısıtlamalar                              |
|---------------------|-----------------|-------------------------------------------|
| id                  | Long (IDENTITY) | PK, Auto-increment                        |
| accountNumber       | String(10)      | UNIQUE, NOT NULL (Örn: 1029384756)        |
| iban                | String(26)      | UNIQUE, NOT NULL (Örn: TR12000990000...)  |
| balance             | BigDecimal      | NOT NULL, DEFAULT=0                       |
| currency            | Enum (STRING)   | NOT NULL (TRY / USD / EUR)                |
| ownerIdentityNumber | String(11)      | NOT NULL (TC veya Vergi No)               |
| isActive            | boolean         | NOT NULL, DEFAULT=true (Soft Delete)      |

İlişkiler:
- Account -> sentTransactions    (OneToMany, mappedBy=senderAccount)
- Account -> receivedTransactions (OneToMany, mappedBy=receiverAccount)

### Tablo: transactions
| Kolon           | Tip             | Kısıtlamalar                                     |
|-----------------|-----------------|--------------------------------------------------|
| id              | Long (IDENTITY) | PK, Auto-increment                               |
| referenceNo     | String          | UNIQUE, NOT NULL, updatable=false (UUID)         |
| sender_account_id | Long          | FK -> accounts.id, nullable (ATM yatırmada null) |
| receiver_account_id | Long        | FK -> accounts.id, nullable (ATM çekmede null)  |
| amount          | BigDecimal      | NOT NULL                                         |
| convertedAmount | BigDecimal      | nullable (farklı döviz transferlerinde kullanılır)|
| transactionType | Enum (STRING)   | NOT NULL                                         |
| status          | Enum (STRING)   | NOT NULL, DEFAULT='COMPLETED'                    |
| description     | String(250)     | nullable                                         |
| transactionDate | LocalDateTime   | NOT NULL, @CreationTimestamp (otomatik)          |

TransactionType Enum: DEPOSIT, WITHDRAWAL, TRANSFER, SALARY, BILL_PAYMENT
TransactionStatus Enum: COMPLETED, PENDING_APPROVAL, REJECTED

### Enum: BillType
ELECTRICITY, WATER, GAS, INTERNET, TELEPHONE

## 4. GİRDİ VE ÇIKTILAR (DTO İÇERİKLERİ)

### Request DTO'ları
CreateAccountRequest:
  - currency : Account.Currency (@NotNull, TRY/USD/EUR)

DepositRequest:
  - iban   : String (@NotBlank)
  - amount : BigDecimal (@NotNull, @DecimalMin="1.0")

TransferRequest:
  - senderIban   : String (@NotBlank)
  - receiverIban : String (@NotBlank)
  - amount       : BigDecimal (@NotNull, @DecimalMin="1.0")
  - description  : String (opsiyonel)
  - isSalaryPayment : Boolean (maaş ödemesinde MASAK limitini bypass eder)

BulkSalaryRequest:
  - senderIban  : String (şirketin çıkış IBAN'ı)
  - companyName : String (dekont açıklaması için)
  - salaryItems : List<SalaryPaymentItem>

SalaryPaymentItem:
  - receiverIban : String (personelin IBAN'ı)
  - amount       : BigDecimal (TRY cinsinden maaş tutarı)

InternalPaymentRequest (bill-service'ten gelir):
  - accountId   : Long
  - amount      : BigDecimal (TRY cinsinden fatura borcu)
  - description : String

OpenAccountRequest (Admin için):
  - currency : String ("TRY", "USD", "EUR")

### Response DTO'ları
AccountResponse:
  - id             : Long
  - accountNumber  : String
  - iban           : String
  - balance        : BigDecimal
  - currency       : Account.Currency
  - isActive       : boolean
  - ownerName      : String (placeholder: "Müşteri (***1234)")
  - identityNumber : String

TransactionResponse:
  - referenceNo      : String (UUID dekont no)
  - amount           : BigDecimal
  - convertedAmount  : BigDecimal
  - transactionType  : TransactionType (DEPOSIT/WITHDRAWAL/TRANSFER/SALARY/BILL_PAYMENT)
  - status           : TransactionStatus (COMPLETED/PENDING_APPROVAL/REJECTED)
  - description      : String
  - transactionDate  : LocalDateTime
  - senderAccountId  : Long (nullable)
  - receiverAccountId: Long (nullable)

AccountValidationResponse:
  - ownerIdentityNumber : String (TC No)
  - isActive            : boolean

SystemLogResponse:
  - timestamp  : String
  - level      : String (INFO/WARN/ERROR)
  - appName    : String
  - threadName : String
  - loggerName : String
  - message    : String

ExchangeRateResponse:
  - base_code : String (Örn: "TRY")
  - rates     : Map<String, Double> (Örn: {"USD": 0.03, "EUR": 0.028})

## 6. İŞ KURALLARI (BUSINESS LOGIC)

### Hesap Açma
- AccountUtils.generateAccountNumber(): 10 haneli rastgele sayı (1000000000-9999999999)
- AccountUtils.generateIban(): "TR" + 2 haneli doğrulama + "00099" + "0" + "000000" + hesapNo (26 karakter)
- Çakışma koruması: Üretilen numara/IBAN DB'de varsa tekrar üret (do-while döngüsü)
- Hesap açıldığında bakiye = 0, isActive = true

### Hesap Silme (Soft Delete)
- Bakiye > 0 ise BankOperationException fırlatılır ("İçerisinde bakiye bulunan hesap kapatılamaz")
- Admin rolü: Başkasının hesabını da kapatabilir
- Müşteri rolü: Sadece kendi hesabını kapatabilir
- Kapatma işlemi: isActive = false yapılır (Hard Delete YOK, geçmiş dekontlar korunur)

### Transfer İş Kuralları
1. Gönderen hesap sahibi, işlem yapan kişi ile aynı olmalı
2. Bakiye yeterli olmalı
3. Aynı IBAN'a transfer yapılamaz
4. Tutar > 0 olmalı
5. Her iki hesap da aktif olmalı
6. **MASAK Kuralı:** isSalaryPayment=false VE TRY cinsinden tutar >= 50.000 TL ise:
   - Bakiye gönderenden ANINDA düşülür
   - Alıcıya para GEÇMEZ
   - Status = PENDING_APPROVAL
   - Admin'e SYSTEM_ALERT gönderilir
7. Döviz transferinde: CurrencyServiceClient üzerinden anlık kur çekilir (Cache bypass)

### MASAK Onaylama (Admin Approve)
- Status PENDING_APPROVAL olmalı, değilse hata
- Alıcı hesabın bakiyesi convertedAmount kadar artırılır
- Status = COMPLETED, description'a "[ONAYLANDI]" eklenir

### MASAK Reddetme (Admin Reject)
- Status PENDING_APPROVAL olmalı, değilse hata
- Gönderenin bakiyesi orijinal amount kadar geri iade edilir
- Status = REJECTED, description'a "[REDDEDİLDİ VE İADE EDİLDİ]" eklenir

### Toplu Maaş (Internal - Corporate'ten gelir)
1. Şirketin çıkış kasası bulunur ve aktiflik kontrolü yapılır
2. Toplam maaş toplamı hesaplanır (TRY cinsinden)
3. Şirket kasası yabancı dövizse FeignClient ile TRY->Döviz çevirimi yapılır
4. Bakiye yeterliliği tek seferde kontrol edilir (fail-fast)
5. Her personel için ayrı transfer işlemi çalıştırılır (isSalaryPayment=true -> MASAK bypass)
6. X-Internal-Identity header'ı SecurityContextHolder'a set edilir

### Müşteri Hesap Silme (Internal - Auth'tan gelir)
- Müşterinin tüm hesapları çekilir
- Herhangi bir hesapta bakiye > 0 ise BankOperationException ("Bakiyesi olan hesaplar var")
- Bakiye sıfırsa tüm hesaplar isActive=false yapılır (saveAll ile toplu güncelleme)

### Fatura Tahsilatı (Internal - Bill Service'ten gelir)
- Hesap accountId ile bulunur, sahiplik kontrolü yapılır (identityNumber ile)
- Fatura TRY cinsinden; hesap yabancı dövizse TRY->Döviz çevirimi yapılır
- Bakiye yeterli değilse BankOperationException
- BILL_PAYMENT tipinde transaction kaydedilir

### Elasticsearch Log Sorgulama
- ElasticsearchClient (FeignClient) ile POST /bank-logs-*/_search sorgusu
- Limit ve level (INFO/WARN/ERROR/ALL) parametreleriyle filtrelenebilir
- Sıralama: @timestamp DESC
- Yanıt: hits->hits->_source alanından parse edilir
- Hata durumunda boş liste döner (exception yutulur, log atılır)

### Güvenlik
- JwtAuthenticationFilter: X-Identity-Number header'ından TC okunur, SecurityContextHolder'a set edilir
- InternalHeaderFilter: İç servislerden gelen isteklerde header doğrulaması
- Hesap sahipliği her kritik işlemde SecurityContextHolder.getAuthentication().getName() ile kontrol edilir
