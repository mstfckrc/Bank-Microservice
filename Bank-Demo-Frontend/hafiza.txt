# 🖥️ ABSOLUTE BLUEPRINT: Bank-Demo-Frontend - "KULLANICI VİTRİNY"

### 1. Servisin Amacı
Son kullanıcıların (Müşteri, Şirket Yöneticisi, Sistem Yöneticisi) banka sistemine eriştiği grafiksel arayüzdür. React ve Next.js (App Router) ile inşa edilmiştir. Verileri doğrudan API Gateway (`port: 8081`) üzerinden çeker. Kendi veritabanı yoktur.

---

### 2. Teknoloji Yığını (Tech Stack)
*   **Framework:** Next.js (v14/v15 App Router Mimarisi)
*   **Ağ Katmanı:** Axios (API Gateway ile iletişim)
*   **Durum Yönetimi (State):** Zustand (`useAuthStore.ts`)
*   **Tasarım Sistemi:** Tailwind CSS, Shadcn UI, Radix UI
*   **Güvenlik (Captcha):** Google reCAPTCHA v2 (`react-google-recaptcha`)
*   **Kimlik Doğrulama:** NextAuth.js kullanılmamış, tamamen manuel Cookie tabanlı JWT süzgeci tasarlanmıştır.

---

### 3. Klasör ve Rota Yapısı (App Router)
Görünümler ve paneller "Route Groups" özelliğiyle `( )` parantez içinde izole edilmiştir.

*   **`(auth)`**: Giriş ve Kayıt olma ekranları. (`/login`, `/register`)
*   **`(dashboard)`**: Sisteme giriş yapıldıktan sonraki ana paneller. Üç farklı dünyaya ayrılır:
    *   `/user/dashboard` -> Bireysel Müşteriler (Havale, Fatura, Hesaplar).
    *   `/company/dashboard` -> Kurumsal Yöneticiler (Personel ekleme, Toplu Maaş).
    *   `/admin/dashboard` -> Sistem Yöneticileri (MASAK Onayları, Log izleme, Müşteri silme).
*   **`services/`**: API Gateway uç noktalarına atılan tüm Axios isteklerinin toplandığı sınıf/fonksiyonlar dizini (`admin.service.ts`, `transaction.service.ts` vb.).
*   **`store/`**: Zustand kullanılarak kullanıcının "Giriş Yapıldı" (Logged In) statüsünün RAM'de tutulduğu yer (`useAuthStore.ts`).

---

### 4. Güvenlik ve Kalkanlar (proxy.ts / Middleware)
Sistemin en hayati Frontend kalkanı olan Middleware (`proxy.ts` veya `middleware.ts`) iki aşamalı bir duvar örer:

1.  **Token Ölüm Kontrolü (Expiration Check):** Tarayıcıdaki `token` çerezini alır, Base64 ile decode edip (JWT'yi manuel parçalayarak) `exp` değerine bakar. Eğer süre dolmuşsa (`exp < Date.now()`), arka plana hiç istek atmadan doğrudan Keycloak'un Logout URL'sine (`/protocol/openid-connect/logout`) yönlendirme yapar ve çerezleri siler. Çıkış işlemini kusursuzlaştırmak için `id_token_hint` kullanır.
2.  **Yetki Duvarları (Role-Based Access):**
    *   `/admin` rotalarına girmek isteyenlerin token'ında `"ADMIN"` rolü yoksa `/login` ekranına atar.
    *   `/company` rotalarına girmek isteyenlerin `"CORPORATE_MANAGER"` rolü yoksa bireysel panele atar.
    *   Oturum açmış (Token'ı olan) birisi `/login` sayfasına gitmeye çalışırsa, onu sahip olduğu role göre kendi kontrol paneline (Dashboard) geri fırlatır.

---

### 5. Çevre Değişkenleri (.env.local)
*   `NEXT_PUBLIC_API_URL`: API Gateway adresi (`http://localhost:8081/api/v1`).
*   `NEXT_PUBLIC_KEYCLOAK_URL`: Keycloak kimlik doğrulama ekranının ana urli (`http://localhost:9090`).
*   `KEYCLOAK_CLIENT_ID` / `SECRET`: Frontend'in Keycloak'a kendini tanıtması için gerekli gizli anahtarlar.

---

### 6. Temel İş Kuralları
1.  **Vitrin Kurları:** Ana sayfada veya döviz ekranında `currency-service`'in sunduğu `/api/v1/currencies/rates` uç noktasından alınan veriler gösterilir. Bu veriler Gateway üzerinden geçerken Auth gerektirmez (Sivil Bölge).
2.  **Axios Interceptor Mantığı (Muhtemel):** Tüm hizmetler (`services/*.ts`), kullanıcı çerezindeki JWT'yi alıp Gateway'e `Authorization: Bearer <token>` başlığıyla göndermek zorundadır. Aksi takdirde Gateway'in Askeri Bölge filtresinden geçemezler.
