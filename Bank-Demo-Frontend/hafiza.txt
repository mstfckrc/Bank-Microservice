# 💻 SEC-DEMO FRONTEND HAFIZA KARTI (V2.1 - ULTRA DETAYLI & SIKIŞTIRILMIŞ)

Yapay zekanın (veya yeni bir geliştiricinin) arayüz kodlarına tek bakışta %100 hakim olması için tasarlanmış yapısal haritadır. Gereksiz metinler yerine **"Bileşen - İşlev - Beklenti"** formatında sıkıştırılmıştır.

---

## 🗺️ 1. MİMARİ VE ROTA (ROUTING) HARİTASI
*   **Çekirdek Yığın:** Next.js 14+ (App Router), React Server Components, TypeScript, TailwindCSS, Shadcn UI.
*   **Güvenlik Muhafızı (`proxy.ts` / Middleware):** Tüm istekler buradan geçer. Token yokken korumalı rotaya (örn: `/user`) girilirse `/login`'e yönlendirir. Rol kısıtlamaları aktiftir (**Strict Routing**). (Token çözümlemesi `Buffer` ile yapılır).
*   **Mekansal Dağılım:**
    *   `/(auth)/*`: Sadece tokenı olmayanların görebileceği giriş ve kayıt ekranları.
    *   `/currencies`: Tüm ziyaretçilerin güncel kurları takip edebildiği canlı döviz tahtası.
    *   `/(dashboard)/*`: `Navbar` ve `Sidebar` içeren ortak çerçeve.
        *   `/user/dashboard`: Bireysel Müşterilerin cüzdanları, döviz transferleri, fatura işlemleri.
        *   `/company/dashboard`: Kurumsal Yöneticilerin iş gücü yönetimi, toplu maaş ödemeleri ve **Otomatik Maaş Ayarları**.
        *   `/admin/dashboard`: God-Mode yetkili müşteri/hesap döküm ekranları.

---

## 🧠 2. MERKEZİ HAFIZA (STATE) VE TİPLER (types/index.ts)
Frontend tarafı, backend'den gelen JSON'ı kesinlikle hata payı olmadan bu tiplere bağlar.

| Tür/Kategori | İçerik ve Mantık (Logic) |
| :--- | :--- |
| **`useAuthStore`** | Zustand mağazası. State: `user`, `token`, `isAuthenticated`. Metotlar: `login()`, `logout()`, `updateUser()`. |
| **Auth & Profil** | `LoginRequest` (identityNumber, password), `RegisterRequest` (Role bağlamlı firstName/companyName), `UserProfileResponse`. |
| **Hesap (Account)** | `AccountResponse` (Müşteri hesap bilgisi, `isActive` durumu kontrolü gerekir). |
| **İşlem (Transaction)**| `TransactionResponse` (Yeşil/Kırmızı bakiye hesabı frontend'de `senderAccountId` / `receiverAccountId` ilişkisine göre render edilir). |
| **Kurumsal Yön.** | `CompanyEmployeeResponse`, `HireEmployeeRequest`, `SalaryPaymentRequest` (Toplu maaş), **`AutoPaymentSettingsRequest`** (Yeni: Otomatik ödeme günleri). |
| **Döviz / Fatura** | `ExchangeRateResponse` (Live kurlar), `BillInstructionRequest` / `Response` (Elektrik/Su oto. fatura). |

---

## 🔌 3. SERVİSLER VE CUSTOM HOOKS (API KATMANI)
İstekler `services/` içindedir, UI tarafında `hooks/` içinden (React Hook formunda) çağırılıp kullanılır. Component içine doğrudan `axios` YASAKTIR.

*   **`useAccounts.ts`:** Kullanıcının cüzdanlarını tutar.
*   **`useCompanyEmployees.ts`:** Personelleri (`fetchEmployees`), işe alımı, işten çıkarmayı yönetir. Optimistic UI eğilimindedir.
*   **`useSettings.ts` (Yeni):** Kurumsalın oto-ödeme ve maaş günlerini çeker / günceller.
*   **`useBills.ts` & `useCustomers.ts` & `useUserDashboard.ts`:** Dashboard'daki veri akışlarını izole eder.
*   **Not:** API İsteklerinde `BASE_URL` çiftleme hatası (`/api/v1/api/v1/...`) olmaması için env ve base url ayrımlarına dikkat edilir. E.g: Sadece `/customers` yazılıp service çağırılır.

## 🚨 5. YAPAY ZEKA VE GELİŞTİRİCİLER İÇİN SIKI KURALLAR
1. **Naming Convention (İsimlendirme):** İsteklerde `tcNo` gibi değişkenler **asla** kullanılmaz. Uç nokta (endpoint) veya JSON property adı her zaman `identityNumber`'dır.
2. **Double Path (URL Bug):** Axios base_url `.env` ile eşleştiğinden servis dosyalarında sadece hedefi (`/accounts` gibi) ver.
3. **Modüllere Ekleme Yaparken:** Eğer Kredi Kartı gibi yeni modül istenirse -> 1. `types/index.ts` içine tipleri yaz. 2. `services/` içine `.service.ts` dosyasını oluştur. 3. Hook'unu yaz. 4. Component arayüzünü oluştur. Kısayol yasaktır.
4. **Token Decoding:** Tarayıcı tarafındaki her "giriş yapılmış mı" veya "rol ne" sorgusunda JWT parse edilirken Base64/Buffer kısımlarına ve proxy middleware'lere güvenilir, manuel local storage regexleri kullanılmaz.
